# Track: Onboarding UI & Provisioning Flow
**Created**: 2026-03-19
**Status**: planning
**Priority**: high

## Summary
Implement the 3 core screens for the Onboarding Flow (Register, Industry Selection, Provisioning Loader) and wire them up with the `POST /api/v1/auth/register` and `POST /api/v1/tenants/provision` APIs. Convert the static HTML/Tailwind mockups generated by StitchMCP into proper Next.js App Router components using Lucide icons, `react-hook-form`, and the izhubs `AppLayout` styling.

## User Stories
- As a new user, I want to register and choose my industry so that my workspace is tailored to my business model.
- As a user, I want to see a loading screen during setup so that I know my workspace is being prepared.
- As an admin, I want initial demo data generated so my dashboard isn't empty upon first login.

## Acceptance Criteria
- [ ] Màn hình `/register` hoạt động với react-hook-form và Zod.
- [ ] Giao diện Onboarding Wizard chuyển đổi mượt mà giữa các ngành.
- [ ] API `POST /api/v1/tenants/provision` copy đúng `theme_defaults` và `nav_config` từ `industry_templates`.
- [ ] Tự động chuyển hướng về `/dashboard` sau khi provision xong.

## Technical Plan

### DB Changes
- Phía DB đã có sẵn các bảng `tenants`, `users`, `industry_templates` từ `001_initial_schema.sql`.
- Không cần thêm Table mới, tập trung vào API thao tác DB.

### API Endpoints
| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/api/v1/tenants/provision` | requireAuth | Thực thi kích hoạt module, copy theme/config và nhúng Seed Data. |

### UI Components & Pages
- Template/Layout: `app/(onboarding)/layout.tsx` (Minimal UI).
- Page: `app/(auth)/register/page.tsx`
- Page: `app/(onboarding)/onboarding/page.tsx`
- Page: `app/(onboarding)/provisioning/page.tsx`

### Engine Functions
- `core/engine/tenants.ts`: hàm `provisionTenantBackgroundJob()` thực thi ngầm việc copy template và cấy demo data.
